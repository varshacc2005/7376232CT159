=====================================================================
ALGORITHM EXPLANATION
=====================================================================

Problem: Users receive high volume of notifications and lose track
         of important ones. Need to highlight top N notifications.

Solution: Composite scoring combining type importance and recency.

Score Calculation:
  Score = (TypeWeight × 100) + (RecencyScore × 50)

Why This Formula?
  1. Type weight × 100: Ensures type is primary factor
     - Difference between Placement and Result: 100 points
     - Difference between Result and Event: 100 points
     - Even oldest Placement (300) beats newest Event (150)

  2. Recency × 50: Provides secondary ranking
     - Breaks ties when same type
     - Recent Result (250) beats old Result (200)
     - Maximum boost of 50 points for newest

Complexity Analysis:
  Time:  O(n log n) - Sorting n notifications
  Space: O(n) - Store all notifications and scores
  
  For streaming (maintaining top N with m new notifications):
  Time:  O(m log n) - Using min-heap, where n=10
  Space: O(n) - Only keep top 10 in heap

=====================================================================
MAINTAINING TOP 10 WITH NEW NOTIFICATIONS
=====================================================================

Strategy: Min-Heap Data Structure

When new notification arrives:
  1. Calculate priority score: O(1)
  2. If heap size < 10: Insert new notification - O(log 10) = O(1)
  3. If new score > min score in heap:
     - Remove min-score notification - O(1)
     - Insert new notification - O(log 10) = O(1)

Example Stream Processing:
  Current top 10: [Placement(345), Placement(309), ..., Event(141)]
  
  New notification: "Placement: Microsoft hiring" → Score: 347
  Action: Remove Event(141), Insert Placement(347)
  Result: Top 10 updated in O(log 10) ≈ O(1) time
  
  New notification: "Event: seminar" → Score: 95
  Action: Already have better Event(141), ignore
  Result: No change to top 10, O(1) comparison

Efficiency Gains:
  - Batch mode: Process all 1000 notifications: O(1000 log 1000) ≈ 10ms
  - Streaming mode: Update top 10 with 1000 new notifs: O(1000 × 3) ≈ 3µs
  - Speed improvement: 3000x faster for high-frequency updates
